Resylib C++11 with Doxygen Dokumentation

Im Rahmen des Projektes „Deeply Embedded“ entstand die Aufgabe zur Entwicklung eine C++ 11 Bibliothek für eingebettete reaktive Systeme. Die Bibliothek ist getestet auf einer Beaglebone Black (Abkürzung BBB).  Das Ziel ist die GPIO- Anbindungen zu Beaglebone Black zur Verfügung zu stellen.

Library for reactive systems: Resylib https://github.com/deeplyembedded

Die Arbeit ist bereits begonnen in Jahr 2015 unter der Leitung des Professors Stephan Pareigis Hochschule für Angewandte Wissenschaften Hamburg. Wir die Gruppe SS16 haben das Projekt 2016 weitergeführt.

Um eine Übersicht über das Projekt zu bekommen, wurde dieses Dokument erstellt.

Durch Klicken  auf die Schaltflächen oben auf der Leiste „Namensbereiche“ „Klassen“ und „Dateien“ gelangen Sie zu den unterschiedlichen Bereichen der Dokumentation. Zudem sind Zusammenhänge zwischen Klassen und deren Instanzen (Vererbungshierarchie) und Abhängigkeiten zwischen Methoden dargestellt.

Diese Dokumentation wurde generiert durch Doxygen in HTML Format.

Doxygen ist ein freies Werkzeug zur Softwaredokumentation. Durch spezielle Kommentare im Quellcode, erstellt Doxygen eine Dokumentation in HTML und auch andere Dateiformaten wie LaTeX, XML,  PostScript PDF und Manpages. Dabei analysiert Doxygen sowohl die Kommentare, als auch den Quellcode selbst, um Module, Klassen, Methoden und Funktionen sowie verschiedene Arten von Variablen zu erkennen und zu dokumentieren.

Hardware-Funktionen wurden in Resylib implementiert

Actor erhält Nachrichten von BeagleBone Black und führt dann verschiedene Reaktionen durch. Hier Beispiele:

  • LED

Öffentliche Funktionen von LED mit Digital-Output

  • LED (const GPIOPin pin);

Gewünschte Digital-GPIO-Pin von LED wird aktiv

  • void on();

macht LED On

  • void off(); 

macht LED Off

  • void toggle(); 

Wenn On,dann Off.  Oder wenn Off, dann On

  • ~LED();

Instanz von LED ist abgeschlossen

  • Servo

                                             

Öffentliche Funktionen Servo angesteuert über PWM

  • Servo(GPIOPin servoPin); 

Instanz des Servos wird erstellt und erzeugt automatisch GPIO-PWM.

  • void enableServo();

Das GPIO-PWM wird aktiviert, also Servo ist aktiv

  • void setPosition(double position);

Servo ändert sich die Position (Wert: min. 0.0 bis max. 1.0)

  • double getPosition(void);

liest die aktuelle Servo-Position (Wert: min. 0.0 bis max. 1.0)

  • void disableServo();

Servo deaktivieren

Sensor sendet Signal von der verschiedenen Reaktion zum BeagleBone Black, also bekommt das BBB Information. Hier Beispiele:

  • Einpoliger Schalter:

                                   

Öffentliche Funktionen von einpoligem Schalter mit Digital-Input.

  • Switch();

GPIO Digital Input von Schalter wird aktiv

  • void initialize(GPIOPin pin_); 

Ein gewünschter GPIO Pin als Parameter übergeben, dann wird GPIO-Digital mit Input aktiviert.

  • bool isPressed();

liest, ob der Schalter in einem aktiven Zustand ist.

  • void setActiveState(GPIO_Digital::State state);

Der aktive Zustand des Schalters auf HIGH oder LOW wird eingegeben.

  • virtual ~Switch();

löscht Instanz von dem Schalter.

  • KY-023 Joystick:

                               

Öffentliche Funktionen von Joystick mit Digital-Input

  • JoystickDigital();

Einfach Instanz von Joystick wird erzeugt.

  • void initialize(GPIOPin pinX_, GPIOPin pinY_);

 Zwei gewünschte GPIO-Pins (für X- und Y-Richtung) belegen, dann als Input-Pins aktivieren

  • void initialize(GPIOPin pinX_, GPIOPin pinY_, GPIOPin pinSW_);

 ist gleich wie oben Funktion mit einem einpoligen Schalter.

  • Switch& getSwitchObject();

liest, ob der Schalter in einem aktiven Zustand ist.

  • JoystickDigital::JoystickDirection direction();

liest den Zustand von Joysticks. Seine Rückgabe: NORTH_EAST, SOUTH_EAST, SOUTH_WEST, NORTH_WEST oder UNKNOWN.

  • virtual ~JoystickDigital();

löscht Instanz vom Joystick.

Hardware Manager (Abkürzung: HWManager)

Es ist wie Dispatcher, er steuert GPIO Pins (Sensoren und Aktoren) und hat nur eine Instanz (Singleton).

Seine Aufgaben:

  • std::unique_ptr<GPIO> createGPIOResource(...);

erzeugt GPIO Ressource (ZB: Output-Digital,  Input-Digital oder PWM)

  • void allocateGPIOPin(const RSL::GPIOPin pin);

addiert eine neue GPIO Pin (sie wird in Speicher zugeordnet)

                 

  • void freeGPIOPin(const RSL::GPIOPin pin);

löscht ausgewählte GPIO Pin

Noch anstehende Aufgaben in Hardware Manager:

  • Observer pattern (das Beobachter-Muster). ZB: Wenn Aktoren (als Observer) in HWManager (als Subjekt) registriert werden, erhält jeder der Aktoren dann automatisch Update durch Reaktion von HWManager.
