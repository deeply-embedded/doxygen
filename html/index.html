Resylib mit Doxygen

Im Rahmen des Projektes „Deeply Embedded“ entstand die Aufgabe zur Entwicklung eine C++ 11 Bibliothek für eingebettete reaktive Systeme. Die Bibliothek ist getestet auf einer Beaglebone Black (Abkürzung BBB).  Das Ziel ist die GPIO- Anbindungen zu Beaglebone Black zur Verfügung zu stellen.

 

Library for reactive systems: Resylib https://github.com/deeplyembedded

Die Arbeit ist bereits begonnen in Jahr 2015 unter der Leitung des Professors Stephan Pareigis Hochschule für Angewandte Wissenschaften Hamburg. Wir die Gruppe SS16 haben das Projekt 2016 weitergeführt.

Aus der Projektarbeit und dem Quellcode der Projektgruppen wurde ein Dokument erstellt. Unter diesem Link https://deeply-embedded.github.io/doxygen/html/index.html  finden Sie eine übersichtliche Dokumentation der Dateien, Klassen und Funktionen der Projektarbeit.  Diese Dokumentation wurde generiert durch Doxygen in HTML Format.

Doxygen ist ein freies Werkzeug zur Softwaredokumentation. Durch spezielle Kommentare im Quellcode, erstellt Doxygen eine Dokumentation in HTML und auch andere Dateiformaten wie LaTeX, XML,  PostScript PDF und Manpages. Dabei analysiert Doxygen sowohl die Kommentare, als auch den Quellcode selbst, um Module, Klassen, Methoden und Funktionen sowie verschiedene Arten von Variablen zu erkennen und zu dokumentieren.

Aus der Aufgabenstellung hat die Projektgruppe SS16 ein vierbeiniger Roboter als Anwendungsbeispiel ausgewählt um BeagleBone Anwendungen in Kommunikation, Steuerung und Programmierung C++ zu erkunden und zu testen.

Hardware-Funktionen wurden in Resylib implementiert

Actor enthaltet Signal wie Nachricht aus BeagleBone Black und führt dann verschiedene Reaktionen durch. Hier Beispiele:

  • LED:

Öffentliche Funktionen von LED mit Digital-Output

  • LED (const GPIOPin pin); //Gewünschte Digital-GPIO-Pin von LED wird aktiv
  • void on();// macht LED On
  • void off(); // macht LED Off
  • void toggle(); // Wenn On,dann Off.  Oder wenn Off, dann On
  • ~LED(); //Instanz von LED ist abgeschlossen

  • Servo

Öffentliche Funktionen Servo angesteuert über PWM

  • Servo(GPIOPin servoPin); //Instanz des Servos wird erstellt und erzeugt automatisch GPIO-PWM.
  • void enableServo(); //das GPIO-PWM wird aktiviert, also Servo ist aktiv
  • void setPosition(double position); //Servo ändert sich die Position (Wert: min. 0.0 bis max. 1.0)
  • double getPosition(void); //liest die aktuelle Servo-Position (Wert: min. 0.0 bis max. 1.0)
  • void disableServo();// Servo deaktivieren

Sensor sendet Signal von der verschiedenen Reaktion zum BeagleBone Black, also bekommt das BBB Information. Hier Beispiele:

  • Einpoliger Schalter:

Wichtige öffentliche Funktionen von einpoligem Schalter mit Digital-Input.

  • Switch(); //GPIO Digital Input von Schalter wird aktiv
  • void initialize(GPIOPin pin_); // Ein gewünschter GPIO Pin als Parameter übergeben, dann wird GPIO-Digital mit Input aktiviert.
  • bool isPressed(); //liest, ob der Schalter in einem aktiven Zustand ist. (true = An oder false = Aus)
  • void setActiveState(GPIO_Digital::State state); //Der aktive Zustand des Schalters auf HIGH oder LOW wird eingegeben.
  • virtual ~Switch(); //löscht Instanz von dem Schalter.

  • KY-023 Joystick:

Wichtige öffentliche Funktionen von Joystick mit Digital-Input

  • JoystickDigital(); //Einfach Instanz von Joystick wird erzeugt.
  • void initialize(GPIOPin pinX_, GPIOPin pinY_); // Zwei gewünschte GPIO-Pins (für X- und Y-Richtung) werden eingegeben, dann werden beide GPIO-Digitals mit Input aktiviert.
  • void initialize(GPIOPin pinX_, GPIOPin pinY_, GPIOPin pinSW_); //gleich wie oben Funktion mit einem einpoligen Schalter.
  • Switch& getSwitchObject(); //liest, ob der Schalter in einem aktiven Zustand ist.
  • JoystickDigital::JoystickDirection direction();//liest, welche Richtung hat der Joystick gezeigt. Sein Rückgabe: NORTH_EAST, SOUTH_EAST, SOUTH_WEST, NORTH_WEST oder UNKNOWN.
  • virtual ~JoystickDigital(); //löscht Instanz vom Joystick.

Hardware Manager (Abkürzung: HWManager) wie Dispatcher, er steuert GPIO Pins (Sensoren und Aktoren) und soll nur ein Instanz (Singleton) bleiben.

Seine Aufgaben:

  • std::unique_ptr<GPIO> createGPIOResource(...);

erzeugt GPIO Ressource (ZB: Output-Digital,  Input-Digital oder PWM)

  • void allocateGPIOPin(const RSL::GPIOPin pin);

addiert eine neue GPIO Pin (sie wird in Speicher zugeordnet)

                     

  • void freeGPIOPin(const RSL::GPIOPin pin);

löscht ausgewählte GPIO Pin

Noch anstehende Aufgaben in Hardware Manager:

  • Observer pattern (das Beobachter-Muster): ZB Wenn Aktoren (als Observer) in HWManager (als Subjekt) registriert werden, erhält jeder der Aktoren  automatisch Update durch Reaktion von HWManager.
